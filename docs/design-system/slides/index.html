Before · After

From scattered files to a single portal

The same dataset, before and after running through a Niamoto project. Nothing is generated or invented — the pipeline only structures what already exists.

Before

A folder of files

Seven years of occurrences, three shapefiles, and a working spreadsheet that only one person can read.

  • 16 files · 4 formats · no shared vocabulary
  • Taxon names spelled four different ways
  • No pipeline — each update is a manual merge
2–3 weeksto prepare a single report
After

A versioned portal

The same data, loaded once, enriched by plugins, then rendered as a browsable biodiversity portal you can rebuild on demand.

  • One SQLite · four collections · shared schema
  • Names matched against World Flora Online
  • niamoto publish · 2.4s to rebuild
2.4 secondsto rebuild the full site
